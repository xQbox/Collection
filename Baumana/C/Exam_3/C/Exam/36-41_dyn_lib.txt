Вопрос 36-41 ( Динамические библиотеки )
	- Библиотека - это заголовочный файл плюс откомпилированный файл этой библиотеки.
		( двоичный код предотвращает доступ к исходному коду самой библиотеки )

Теория:	
		- Статические библиотеки. 
			- Связываются  с программой в момент компоновки. 
			- Код библиотеки помещается в исходный файл программы.
				+ Исполняемый файл включает в себя все необходимое. (нет версионности)
				+ Не возникает проблем с некорректной версией.
				
				- Размер (бОльший размер относительно динамической) 
				- При обновлении библиотеки программу нужно пересобрать.
		
		
		- Динамические библиотеки. 
			- Подпрограммы из библиотеки загружаются в приложение во время выполнения программы.
			- Код библиотеки Не помещается в исходный файл программы.
			
				+ Несколько программ могут использовать\разделять одну библиотеку.
				+ Меньшей размер приложения относительно статической библиотеки.
				+ Модернизация библиотеки не требует перекомпиляции программы.
				+ Могут использовать программы на разных языках.
				
				- Требует наличие библиотеки на компьютере.
				- Версионность библиотек. 
				
			- Способы загрузки динамической библиотеки:
				- динамическая компоновка. (компоновщик самостоятельно реализует часть функций для подключения библиотеки)
				- динамическая загрузка.   (приложение берет ответственность на себя за реализацию библиотеки)
			
			
Дополнительно:
		 - Этапы создания библиотек:
		 	- Написаное библиотеки
		 	- Создание объектных файлов библиотеки
